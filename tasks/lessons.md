# Lessons

## 2026-02-23

- 症状：同一功能在不同入口行为不一致时，优先比对入口分支是否存在“平台专用 API 静默失败”。
- 规则：凡是文件读取链路，都必须提供可观测反馈（提示/日志）和可执行回退路径，避免“可选文件但无结果”。
- 症状：Vue `script setup` 中引用未导入常量时，构建可通过但运行时会直接空白/卡顿。
- 规则：涉及常量或函数引用的改动后，必须至少跑一次类型检查或做“符号导入核对”，避免运行时 `ReferenceError`。
- 症状：图标抓取遇到源站 4xx（如 418）时若无冷却机制，会形成同源请求风暴并拖垮 uTools。
- 规则：外部资源抓取必须包含失败冷却与资源释放（尤其是隐藏窗口/浏览器实例），防止卡死。
- 症状：侧栏选中态如果依赖主题变量混合色，容易在深浅色下出现对比不足“看不见选中”。
- 规则：导航选中态优先使用高对比前景/背景配对，并在深色模式单独校准。
- 症状：组件级 focus ring 在桌面插件中会造成统一视觉噪声（黄框/外圈）。
- 规则：若产品要求移除焦点视觉，必须提供一条全局 focus 策略统一覆盖，避免遗漏组件。
- 症状：AI 默认模型只改文案不改调用参数，会出现“显示已改、实际未生效”。
- 规则：模型默认值调整必须同时修改运行时参数与设置页提示文案。
- 症状：用户明确要求“直接给结果”时，如果把主要产出写进 todo 会偏离预期。
- 规则：当用户指定输出方式时，主交付必须直接在回复中完整给出，任务文件仅作辅助记录。
- 症状：用户先讨论方案后明确要求“直接改代码”，若继续停留在讨论会造成来回确认和时间浪费。
- 规则：当用户从讨论切换到执行指令时，立即进入实现与验证流程，讨论结论直接落地为代码。
- 症状：全局仅清理 outline/box-shadow 仍可能残留 Tailwind ring 变量，点击切换时会出现边框闪烁。
- 规则：若产品要求全局隐藏 focus，需同时移除组件级 `focus-visible:ring-*`，并清空 `--tw-ring-*` 变量。
- 症状：UI 库封装按钮可能携带额外状态类，导致“已全局禁焦点但仍闪线”的假象。
- 规则：遇到顽固闪烁时，先把关键交互改成原生按钮并最小样式，隔离封装层影响后再回收样式。
- 症状：全局 `!important` 背景规则容易误伤组件内 hover/active，导致浅色模式“无反馈”但深色模式看似正常。
- 规则：导航/分组交互态优先在组件内定义，避免对通用按钮写无主题条件的全局 `!important` 覆盖。
- 症状：Tailwind 与 UnoCSS 同时生成同名语义类（如 `bg-muted`）时，若 Uno 规则落在后面且写成 `var(--token)`，会让 HSL 片段 token 在浅色模式直接失效。
- 规则：涉及 `--background/--muted/--accent` 等 HSL 片段变量时，关键交互态必须使用 `hsl(var(--token))`（可写在组件样式或任意值类），避免直接依赖同名语义类。
- 症状：仅维护“单一主题配置”会导致深浅模式切换时用户偏好互相覆盖，出现“我改了浅色却影响深色”的感知问题。
- 规则：涉及明暗模式的视觉配置项，优先设计为 `darkConfig + lightConfig` 双通道持久化，并在 UI 中按当前模式展示对应模块。
- 症状：第三方导入文件（如网址精灵）常把分类/书签/笔记混在同一数组，若直接按固定字段读写，容易整批导入失败或静默丢数据。
- 规则：导入功能必须先做“来源识别 -> 统一归一化 -> 边界告警”，并把跳过数量/原因展示给用户。
- 症状：导入后只写 `groups/bookmarks` 不修复 `locations` 和当前选中态，会出现分组计数正常但视图空白或定位错乱。
- 规则：任何导入写入都要在落库前后执行一次“分组-书签关联重建 + activeGroup/activeSub 修正”。
- 症状：导入文件选择器若按后缀过滤（如仅 `.json`），会误伤无后缀但内容合法的导出文件（例如 `data`）。
- 规则：导入入口应允许“任意文件选择 + 内容级格式识别”，不要只依赖扩展名过滤。
- 症状：第三方数据里“回收站/已删除”常用特殊分类 ID（如 `cat@dustbin`），若按普通分类导入会制造伪分组。
- 规则：导入映射要优先识别语义 ID（默认/回收站），并直连到系统内置分组，禁止创建同义新分组。
- 症状：设置页把低频重操作与调试信息直接暴露给普通用户时，常出现“入口变多但主操作更难找”的体验倒退。
- 规则：工具设置默认只保留高频主路径，低频/调试能力应隐藏到开发入口或二级面板，避免在主界面堆叠。
- 症状：统计口径与执行口径分离（例如“缺失数”与“实际匹配目标”来自不同过滤条件）会造成用户感知为功能异常。
- 规则：凡是“数量展示 + 执行动作”成对出现的功能，必须复用同一候选计算函数，并在 store 层集中维护过滤规则。
- 症状：回收站判定若仅依赖 `locations` 且存在提前返回，历史数据索引不一致时会把回收站书签误算进正常统计。
- 规则：涉及“是否在回收站”这类状态判定，必须同时校验 `locations` 与分组索引（`group.children.bookmarkIds`），禁止单一路径短路。
- 症状：设置页使用“快照/覆盖/模式”等技术术语时，普通用户容易误解操作含义，导致不敢点击或误操作。
- 规则：面向终端用户的菜单与按钮文案优先描述“用户能做什么/会发生什么”，避免内部实现词汇。
- 症状：菜单名称与 Toast/提示中的入口名称不一致（如“本地模式”与“本地备份”并存）会让用户按提示找不到路径。
- 规则：重命名设置菜单后，必须同步更新关联提示、通知与引导文案，确保入口名称全链路一致。
