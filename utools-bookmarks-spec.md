# uTools 极简书签插件设计文档

## 目标与基调
- 参考 noice.so 的极简、留白、多层级卡片风格，强调可读性与操作效率。
- 默认字体使用 DM Sans；仅通过 UnoCSS + UnoCSS reset 注入样式，禁止内联 style。
- 图标统一使用 UnoCSS 的现代图标集（如 `@iconify-json/mdi` 等），并支持深浅色自适应。

## 关键需求
- 书签模型：分组（一级）- 子分组（二级）- 书签；书签含标题、描述、URL、图标源、标签、置顶标记。
- 搜索：支持标题/描述/标签/URL 关键词过滤；前端实现，需在列表内高亮匹配词。
- 图标匹配：
  - 创建/编辑时先尝试本机网络直接请求站点 `https://domain/favicon.ico` 或 `<link rel="icon">`，确保使用用户可访问的外网能力。
  - 失败时回退到默认文字图标（首字母圆角背景）；可手动从 UnoCSS 图标集中选择。
  - 缓存命中策略：同域名 24 小时内不重复抓取。
- 布局：
  - 主界面两列模式（≥1280px）/ 单列自适应，卡片展示图标+标题+描述+域名，hover 显示快捷操作（打开/编辑/删除/复制）。
  - 左侧分组树（一级/二级）+ 搜索框；支持折叠、拖拽排序（可选后续迭代）。
- 设置页：
  - 无效地址分析：对选中书签执行 HEAD/GET（最少 1 次，超时 3s），记录状态码与耗时，标记失效。
  - 图标抓取缓存管理：清理/刷新。
  - 导入导出：JSON 结构化文件（含分组关系与图标元数据）。

## 技术栈与框架约束
- Vue 3 + UnoCSS（含 reset），尽量复用组合式函数降低圈复杂度。
- 状态管理：Pinia（持久化到 uTools 本地存储/文件）。
- 组件库：优先使用内置 UI 组件（如 Image 组件），不再引入多余 CSS 方案。
- 网络：使用 `fetch`，遵循用户本机网络环境；增加超时与错误标记，不做冗余 try/catch。

## 数据与存储
- 结构示例（JSON）：
  ```json
  {
    "groups": [
      {
        "id": "g1",
        "name": "设计",
        "children": [
          {
            "id": "g1-1",
            "name": "灵感",
            "bookmarks": [
              {
                "id": "b1",
                "title": "Noice",
                "url": "https://noice.so",
                "desc": "极简动效示例",
                "icon": { "type": "remote", "src": "https://noice.so/favicon.ico" },
                "tags": ["motion"],
                "pinned": false
              }
            ]
          }
        ]
      }
    ]
  }
  ```
- 持久化：首选 uTools db；若需文件导入导出，使用 JSON 文件存放于用户指定目录。

## 交互流程（关键节点）
- 新建书签：输入 URL → 自动抓取标题/描述（可选）与图标 → 用户补充分组/标签 → 保存。
- 搜索过滤：监听输入实时过滤当前分组及其子分组，保持键盘可操作（↑↓选择，Enter 打开）。
- 无效地址检测：批量触发请求，列表内显示状态徽标；可一键移除失效项。

## 主题与动效
- 颜色：使用 CSS 变量 `--bg`, `--fg`, `--muted`, `--accent`，深浅色自动切换；保持对比度 ≥ WCAG AA。
- 动效：卡片 hover 微缩放 + 阴影过渡；分组切换淡入；搜索高亮使用渐变标记。

## 边界与风险
- 境外站点访问失败时需明确显示“使用默认图标/可手动选择”；避免无限重试。
- 大量书签（>1000）时，列表渲染需虚拟化或分页，搜索需节流。
- DM Sans 字体需本地/在线兜底，网络加载失败时退回系统无衬线字体。

## 实施优先级（迭代建议）
1. 核心 CRUD + 分组/二级分组 + 搜索 + 默认图标。
2. 本机网络 favicon 抓取 + 缓存策略。
3. 设置页：无效地址检测 + 导入导出。
4. 拖拽排序 / 键盘导航 / 动效细节优化。

## 验收要点
- 深浅色切换无闪屏；全局仅 UnoCSS 样式生效。
- 搜索、分组、无效检测与图标抓取在离线/弱网情况下具备合理回退。
- 界面与交互保持 noice.so 式极简，信息层级清晰。
